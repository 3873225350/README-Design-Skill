# README Template Library

Use `default.md` for a complete standard README. Use `styles/` for a stronger visual direction.

## Default

- `default.md`: full README homepage skeleton with hero, badges, links, preview, highlights, quick start, usage, structure, roadmap, Star History, Activity Overview, contribution, and license.

## Styles

| Template | Best For | Signature Element |
| --- | --- | --- |
| `01-product-showcase.md` | Apps, browser extensions, SaaS-like tools | Visual hero and screenshots |
| `02-research-roadmap.md` | Research, roadmaps, paper collections | Topic map and output center |
| `03-library-docs.md` | Tools, packages, APIs | Install and 30-second example |
| `04-profile-portfolio.md` | Personal profile READMEs | Featured project portfolio |
| `05-awesome-curation.md` | Curated lists | Category index and selection rules |
| `06-profile-activity-overview.md` | Contributor-focused profiles | Activity Overview contribution chart |
| `07-startup-landing.md` | Product ideas and early startups | Problem, product, why now |
| `08-course-tutorial.md` | Courses and tutorials | Learning map and completion checklist |
| `09-bilingual-global.md` | Chinese projects going global | English/Chinese paired structure |
| `10-minimal-technical.md` | Infra, devtools, serious libraries | Restrained technical clarity |

## Choosing A Template

- Start from `default.md` when the user asks for a "standard" excellent README.
- Use a style template when the repository has a clear identity.
- Combine sections, but do not keep placeholders or false claims.
- Remove Star History for private, new, or internal repositories.
- Remove Activity Overview unless contribution mix is meaningful.
